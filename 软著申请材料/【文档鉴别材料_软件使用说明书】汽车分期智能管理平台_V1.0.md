## 1. 系统概述

### 1.1 开发背景

随着汽车消费金融市场的快速发展，助贷机构在汽车分期业务中承担着越来越重要的角色。传统的人工管理模式存在效率低、流程不透明、数据分散、垫资管理混乱等痛点。本系统针对这些痛点，设计了一套完整的汽车分期业务数字化管理解决方案。

### 1.2 系统定位

本系统是面向汽车分期公司（助贷机构）的业务管理平台，核心管理从"客户进店接单"到"分期还清完结"的全生命周期，同时预留银行审批系统接口，可在未来快速接入银行审批流程。

### 1.3 核心功能

1. **订单全生命周期管理** — 从接单到完结的9大阶段可视化追踪
2. **垫资智能管理** — 申请→审批→出账→计息→还款→逾期监控
3. **GPS实时监控** — 设备管理→定位追踪→分级告警→驾驶舱
4. **资料智能归档** — 标准清单→上传→OCR识别→归档状态追踪
5. **全链路通知引擎** — 9大节点自动触发双端通知
6. **银行审批接口预留** — 标准API设计，快速对接银行系统
7. **数据驾驶舱** — 实时经营数据，老板决策支撑

### 1.4 技术架构

- **后端框架：** Python 3 + FastAPI + Pydantic + SQLite
- **前端框架：** React + Ant Design Pro
- **缓存层：** Redis
- **文件存储：** 阿里云OSS
- **GPS对接：** 博实结/博云车联 API
- **OCR识别：** 阿里云OCR

## 3. 功能模块详解

### 3.1 订单管理模块

**功能描述：** 管理汽车分期订单的全生命周期。

**订单阶段流转：**

```
已接单 → 垫资预审 → 垫资审批中 → 垫资通过 → 垫资已出账 → 垫资已还清
    ↓
银行审批中 → 审批通过 / 审批拒绝
    ↓
放款通知 → 待提车 → 已提车 → GPS安装中 → GPS已在线
    ↓
资料归档中 → 归档完成 → 抵押登记中 → 已抵押
    ↓
正常还款中 → 逾期 → 已结清 → 已完结
```

**核心实体：**

- 客户信息（姓名、手机、身份证、地址）
- 车辆信息（品牌、车型、车架号、价格）
- 贷款信息（首付、贷款金额、期数、月供利率）

### 3.2 垫资管理模块（核心差异化功能）

**功能描述：** 独立管理垫资业务，解决行业内垫资账目混乱的核心痛点。

**垫资单状态机：**

```
待审批 → 审批通过 → 已出账 → 已还清
    ↓        ↓
 审批拒绝   逾期 → 已还清
```

**核心功能：**

1. **垫资申请** — 填写金额、垫资方、利率、起止日期
2. **垫资审批** — 老板/财务审批，记录审批意见
3. **垫资出账** — 记录实际出账时间，更新订单状态
4. **智能计息** — 按日计息，自动计算利息

**计息公式：**

```
日利率 = 月利率 / 30（默认月利率1.5%）
利息 = 垫资本金 × 日利率 × 实际天数
本息合计 = 本金 + 利息
```

**垫资仪表盘：**

展示垫资总余额、今日新垫资、本月新垫资、待还笔数、逾期笔数、近30天垫资余额趋势图。

### 3.3 GPS管理模块

**功能描述：** 集成GPS设备监控，实时掌握抵押车辆位置，防范风险。

**设备类型：**

- 有线GPS — 稳定供电，持续在线
- 无线GPS — 电池供电，定期唤醒
- 隐蔽GPS — 隐藏安装，防拆

**告警类型：**

| 告警类型 | 严重级别 | 说明 |
|---------|---------|------|
| 断电告警 | 🔴 严重 | 设备供电被切断 |
| 拆机告警 | 🔴 严重 | 设备被拆卸 |
| 出区域告警 | 🟡 警告 | 车辆驶出预设区域 |
| 超速告警 | 🟡 警告 | 车辆超速行驶 |
| 低电量告警 | 🟢 提示 | 无线设备电量低 |
| SOS报警 | 🔴 紧急 | 紧急求助信号 |

**GPS驾驶舱：**

实时显示设备总数、在线数、离线数、告警数，支持地图可视化查看车辆实时位置。

### 3.4 资料归档模块

**功能描述：** 对分期业务所需资料进行标准化归档管理，确保合规。

**标准归档清单：**

| 资料类型 | 必需 | 说明 |
|---------|------|------|
| 身份证人像面 | ✅ | OCR识别姓名、身份证号 |
| 身份证国徽面 | ✅ | OCR识别有效期 |
| 行驶证 | ✅ | OCR识别车牌、品牌 |
| 车辆登记证（绿本）| ✅ | OCR识别车架号、抵押状态 |
| GPS安装照片 | ✅ | ≥3张安装实拍 |
| 提车确认单 | ✅ | 双方签字确认 |
| 垫资协议 | ✅ | 垫资双方签署 |
| 购车发票 | ⚠️ | 有则上传 |
| 保险单 | ⚠️ | 有则上传 |

**归档进度流程：**

```
待上传 → 部分上传 → 已完整 → OCR识别中 → OCR已完成 → 已归档
```

### 3.5 通知引擎模块

**功能描述：** 在关键业务节点自动触发通知，确保信息及时传达。

**通知触发节点：**

## 4. 操作流程

### 4.1 标准分期业务流程

**第一步：接单**
1. 销售顾问录入客户信息
2. 录入车辆信息（品牌、型号、价格）
3. 录入贷款方案（首付、贷款额、期数）
4. 系统生成订单号，状态：已接单
5. 客户收到微信通知

**第二步：垫资**
1. 发起垫资申请（公司先替客户垫付购车款）
2. 填写垫资金额、垫资方（公司/个人）、利率
3. 老板/财务审批
4. 审批通过后出账打款
5. 客户收到微信：垫资已到账

**第三步：银行审批**
1. 将客户资料提交银行审批（当前为人工操作）
2. 等待银行结果
3. 录入审批结果（通过/拒绝）
4. 客户收到微信：审批结果通知

**第四步：放款通知**
1. 收到银行放款通知
2. 录入放款信息（金额、日期）
3. 系统自动触发通知：
   - 通知客户提车
   - 通知销售准备陪同
   - 通知贷后准备GPS

**第五步：提车**
1. 填写提车信息（日期、地点）
2. 陪同人员签字确认
3. 状态更新：已提车

**第六步：GPS安装**
1. 安装GPS设备
2. 录入设备编号、IMEI、安装位置
3. GPS平台验证设备在线
4. 系统自动触发：通知接单员收集资料

**第七步：资料归档**
1. 接单员上传全套资料
2. OCR自动识别分类
3. 归档状态实时更新
4. 状态更新：归档完成

**第八步：抵押登记**
1. 在车管所办理抵押
2. 录入抵押信息
3. 设置到期提醒（提前30天）

**第九步：还款监控**
1. 系统按期生成还款计划
2. 还款日提前提醒客户
3. 逾期分级预警（3天/7天/15天）

**第十步：完结**
1. 客户还清全部款项
2. 开具结清证明
3. 办理解押手续
4. 归还绿本给客户

## 5. 核心模块说明

### 5.1 垫资计息引擎

```python
class AdvanceService:
    def calculate_interest(self, principal, rate, start_date, end_date, method="daily"):
        """
        按日计息公式：
        日利率 = 月利率 / 30
        利息 = 本金 × 日利率 × 天数
        """
        days = (end_date - start_date).days
        daily_rate = rate / 30
        interest = principal * daily_rate * days
        return {"days": days, "interest": round(interest, 2)}
```
