# 汽车分期智能管理平台 - 系统设计方案

**版本：** V1.0  
**日期：** 2026-04-02  
**定位：** 面向汽车分期公司老板视角，打造科学、美观、先进的分期业务管理平台

---

## 一、行业现状与痛点分析

### 1.1 市场规模

- 2024年中国汽车金融市场规模超 **2万亿元**，同比增长约10%
- 新车分期 + 二手车分期占主导，助贷模式成为主流
- 银行审批系统（人行征信 + 银行风控）为核心，助贷机构承担获客 + 贷后职责

### 1.2 当前行业痛点

| 痛点 | 现状 | 优化方向 |
|------|------|---------|
| 流程不透明 | 客户不清楚进度，各环节靠电话追问 | 全链路可视化进度追踪 |
| 垫款混乱 | 垫多少、谁垫的、什么时候还，全靠Excel | 独立垫资模块，自动化记账 |
| GPS管理分散 | GPS数据在第三方平台，查不到或查询繁琐 | 集成GPS数据，实时监控大屏 |
| 资料归档乱 | 纸质资料/照片散落，查找困难 | 电子归档系统，OCR识别分类 |
| 放款通知滞后 | 银行通知后靠人工逐一通知客户 | 自动触发通知链，短信+微信 |
| 抵押登记缺失 | 绿本抵押状态不明解押时间不确定 | 抵押状态全周期管理 |
| 跨部门协同差 | 销售、贷后、财务数据不同步 | 统一数据中台，实时共享 |

## 二、竞品分析

| 竞品 | 优点 | 缺点（你的机会） |
|------|------|----------------|
| 互融云 | 功能全，GPS+分期+财务都有 | 界面老旧，垫资模块弱 |
| 泛普ERP | 风控模型完善 | 操作复杂，中小企业用不起 |
| 银行系统（如奇瑞金融） | 数据权威，监管合规 | 仅对银行内部，不面向助贷机构 |
| 传统Excel管理 | 零成本 | 流程不透明、数据分散、无法协同 |
| 乘用车SAAS平台 | 云端、免部署 | 功能标准化，定制差，垫资和GPS管理缺失 |

**你的平台核心差异化：**
1. **垫资透明化** — 行业首创，垫资独立模块
2. **GPS集成驾驶舱** — 实时大屏 + 预警中心
3. **资料智能归档** — OCR自动分类 + 区块链存证（可选）
4. **全链路通知引擎** — 每个节点自动触发客户+员工双端通知

## 四、核心业务流程

### 4.1 全链路业务流

```
客户进店
    │
    ▼
【阶段1：接单】  销售顾问录入客户信息＋车辆信息
    │              ↓
    │         生成订单号，状态：已接单
    │              ↓
    │         客户微信收到：已接单通知
    ▼
【阶段2：垫资预审】（公司内部垫资购车款）
    │              ↓
    │         垫资申请：填写垫资金额、垫资方（公司/个人）、预期还款时间
    │              ↓
    │         财务+老板审批垫资
    │              ↓
    │         垫资打款（状态：垫资中）
    │              ↓
    │         客户微信收到：垫资已出账
    ▼
【阶段3：银行审批】（预留接口，以后可接银行审批系统）
    │              ↓
    │         当前阶段：人工跟进银行审批结果，录入系统
    │              ↓
    │         状态：银行审批中 / 审批通过 / 审批拒绝
    │              ↓
    │         客户微信收到：审批结果通知
    ▼
【阶段4：放款通知】
    │              ↓
    │         收到银行放款通知，录入放款信息
    │              ↓
    │         系统自动触发：
    │         · 通知客户：可以提车了
    │         · 通知销售：准备陪同提车
    │         · 通知贷后：准备GPS安装
    │              ↓
    │         状态：待提车
    ▼
【阶段5：提车】
    │              ↓
    │         填写提车信息（日期、地点、经手人）
    │              ↓
    │         陪同人员签字确认
    │              ↓
    │         状态：已提车，待GPS
    ▼
【阶段6：GPS安装】
    │              ↓
    │         录入GPS设备编号、IMEI号、安装位置
    │              ↓
    │         对接GPS平台，验证设备在线
    │              ↓
    │         GPS状态：在线 / 离线 / 异常
    │              ↓
    │         系统自动触发：
    │         · 通知接单员：GPS已在线，请收集抵押资料
    │              ↓
    │         状态：GPS已装，待资料归档
    ▼
【阶段7：资料归档】
    │              ↓
    │         接单员上传：
    │         · 身份证（人像面+国徽面）
    │         · 行驶证
    │         · 车辆登记证（绿本）
    │         · GPS安装照片
    │         · 提车确认单
    │         · 垫资协议
    │         · 其他补充资料
    │              ↓
    │         OCR自动识别 + 自动分类命名
    │              ↓
    │         状态：资料归档完成，待抵押
    ▼
【阶段8：抵押登记】
    │              ↓
    │         录入抵押登记信息（车管所日期、抵押银行）
    │              ↓
    │         状态：抵押中 / 已抵押
    │              ↓
    │         到期提醒：提前30天推送解押提醒
    ▼
【阶段9：正常还款】
    │              ↓
    │         客户每月还款
    │              ↓
    │         逾期预警：逾期3/7/15天分别触发通知
    ▼
【阶段10：完结】
    │              ↓
    │         客户还清 → 开具结清证明 → 办理解押 → 绿本归还客户
    │              ↓
    │         状态：已完结
    ▼
```

**垫资计息规则：**

```python
# 默认规则（可自定义）
DAILY_INTEREST_RATE = 0.0005   # 日息万分之五（即月息1.5%）
MIN_INTEREST_DAYS = 1          # 最低计息天数

def calculate_interest(advance_amount, start_date, repayment_date, daily_rate=DAILY_INTEREST_RATE):
    days = (repayment_date - start_date).days
    days = max(days, MIN_INTEREST_DAYS)
    interest = advance_amount * daily_rate * days
    return round(interest, 2)
```
